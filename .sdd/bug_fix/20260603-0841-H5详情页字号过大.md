# Bug 修复报告：H5详情页字号过大

## 用户原话

> 我看到服务指引详情页、政策宣传详情页、助企通道详情页里的文字都太大了

## 经验回查

- 是否已有相关经验：是
- 相关经验：
  - `T-003: H5 文章详情 Mock`：文章详情只消费 service 返回的富文本，详情页隐藏 SAFE 入口。
  - `T-004: H5 产品详情 Mock`：产品详情严格展示 7 个业务字段，详情页隐藏 SAFE 入口。
- 为什么仍然犯错：既有经验约束了数据契约和详情页入口行为，但没有把移动端详情页字号纳入可执行验收规则，导致实现沿用了偏大的标题和正文样式。

## 问题分析

- 现象：H5 服务指引详情、政策宣传详情和助企通道详情在手机宽度下文字显得过大。
- 影响范围：
  - `/h5/contents/:id`
  - `/h5/products/:id`
- 直接原因：详情页 CSS 使用了偏大的字号：文章标题 28px、富文本正文 20px、产品标题 32px、产品字段正文 20px 且高字重。
- 根本原因：详情页样式没有按移动端阅读规格收敛，也缺少防止字号回退的测试。

## 修复方案

- 修改文件：
  - `frontend/src/styles/global.css`
  - `frontend/tests/h5-article-detail.spec.ts`
  - `frontend/tests/h5-product-detail.spec.ts`
- 修改说明：
  - 将文章详情标题降至 21px，正文降至 16px，二级标题降至 18px。
  - 将产品详情标题降至 22px，字段标签降至 14px，字段正文降至 16px 并降低字重。
  - 同步收敛详情页顶部栏、图标、卡片内边距和段落间距。
  - 增加测试约束，避免详情页字号回到过大的移动端不适配状态。

## 验证方式

- 前端验证：
  - 打开 `http://localhost:5199/h5/contents/201`，确认服务指引/政策宣传文章详情文字不再过大。
  - 打开 `http://localhost:5199/h5/products/2001`，确认助企通道产品详情文字不再过大。
- 终端验证：
  - `cd frontend`
  - `npm run typecheck`
  - `npm run test`
  - `npm run build`
- 本轮实际结果：
  - typecheck 通过
  - Vitest 6 files / 48 tests passed
  - build 通过
  - 浏览器计算样式确认：文章正文 16px、文章标题 21px、产品字段正文 16px、产品标题 22px
