# Bug 修复报告：H5首页移除加载更多按钮并改为滚动自动加载

## 用户原话

> 不需要加载更多按钮

## 经验回查

- 是否已有相关经验：是
- 相关经验：
  - `T-002: H5 首页栏目与 SAFE 浮动入口 Mock`：H5 首页固定配置收敛到 `src/config/`，页面行为要和 H5 原型/口径一致。
- 为什么仍然犯错：早期 PRD/Plan 明确写过“加载更多”，测试也把该按钮作为验收点；用户验收时产品口径调整后，需要同步更新文档、实现和测试。

## 问题分析

- 现象：H5 首页列表底部展示“加载更多”按钮；移除按钮后若不补充滚动加载，后续分页数据不可见。
- 影响范围：`/h5/` 首页的服务指引、政策宣传、助企通道列表区域。
- 直接原因：`H5LandingPage.vue` 通过 `hasMore` 条件渲染 `.h5-load-more` 按钮，并保留 `loadMore()` 分页加载逻辑。
- 根本原因：旧文档和旧测试仍要求加载更多，未体现用户最新确认的产品口径。

## 修复方案

- 修改文件：
  - `frontend/src/pages/h5/H5LandingPage.vue`
  - `frontend/tests/h5-home.spec.ts`
  - `docs/PRD.md`
  - `docs/Plan.md`
- 修改说明：
  - 移除 H5 首页“加载更多”按钮。
  - 保留分页能力，滚动接近页面底部时自动请求下一页并追加列表数据。
  - 列表首屏内容不足以撑满视口时，自动补拉下一页，避免大屏或短列表场景没有滚动条而无法继续加载。
  - 更新 H5 首页测试，断言页面不再包含“加载更多”和 `.h5-load-more`，并验证滚动后会追加下一页产品。
  - 更新 PRD/Plan，把 H5 首页列表口径改为“不展示加载更多按钮，滚动接近底部自动加载下一页”。

## 验证方式

- 前端验证：
  - 打开 `http://localhost:5199/h5/`，确认底部不再出现“加载更多”按钮。
  - 滚动接近页面底部，确认列表会继续追加下一页数据。
- 终端验证：
  - `cd frontend`
  - `npm run typecheck`
  - `npm run test`
  - `npm run build`
- 本轮实际结果：
  - typecheck 通过
  - Vitest 6 files / 48 tests passed
  - build 通过
  - 浏览器确认 `.h5-load-more` 数量为 0，页面文本不包含“加载更多”；助企通道短页自动补到 12 条，继续滚动后追加到 16 条
