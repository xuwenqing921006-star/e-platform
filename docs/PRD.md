# 央行 E 平台 PRD

> 状态：阶段 A 初稿。页面清单已确认，功能定义、版本规划、关键链路和数据契约将在后续门禁中补充。

## 1. 背景与方案结论
央行 E 平台是一套轻量内容发布平台，包含微信公众号 H5、WEB 管理后台和统一业务服务端。公众通过公众号进入 H5，公开浏览政策内容、县域服务指引和金融产品；后台用户按所属单位和角色维护内容。

已确认采用融合方案：

- H5 首页使用移动端卡片结构，主色为 `#1556D1`。
- 微信公众号 H5 不展示平台名称横幅，页面最顶部直接展示一级 Tab。
- 一级 Tab 使用分段按钮，二级 Tab 使用居中胶囊按钮，乡村振兴三级 Tab 使用文字导航和底部短指示线。
- 内容详情使用正式、克制的长文阅读结构，附件独立下载。

### 3.3 后台权限与功能边界

| 主题 | 包含 | 不包含 |
| --- | --- | --- |
| 管理员 | 管理全部政策、产品、账号和日志 | 无单位绑定 |
| 普通账号 | 仅管理本单位政策内容，修改本人密码 | 不跨单位管理内容，不管理账号和日志 |
| 县域账号 | 仅发布服务指引 | 不发布政策宣传 |
| 产品权限 | 仅管理员和货币信贷政策管理科账号管理产品 | 其他普通账号不显示产品菜单 |
| 内容状态 | 保存后立即展示，删除后立即移除 | 不设置草稿、发布或下架状态 |
| 操作日志 | 记录新增、修改、删除、导入和账号管理等关键动作 | 不做复杂审计分析 |

## 4. Mission、Persona 与版本规划
### 4.1 Mission
通过微信公众号 H5 为公众提供简洁、公开、可读的政策内容与金融产品入口，并通过 WEB 后台让各单位按权限快速维护展示信息。

### 4.2 Persona

| 角色 | 目标 | 主要使用页面 |
| --- | --- | --- |
| 公众访客 | 从公众号快速浏览服务指引、政策宣传、乡村振兴内容和金融产品 | H01、H02、H03 |
| 普通单位账号 | 维护本单位政策内容并修改本人密码 | A01、A02、A03、A04、A12 |
| 县域账号 | 维护本县服务指引并修改本人密码 | A01、A02、A03、A04、A12 |
| 货币信贷政策管理科账号 | 维护本单位政策内容和金融产品 | A01-A08、A12 |
| 管理员 | 统一管理政策、产品、账号和操作日志 | A01-A12 |

### 4.3 V1 / MVP

| 模块 | 包含功能 | 排除功能 | 理由 |
| --- | --- | --- | --- |
| 微信公众号 H5 | 首页、政策详情、产品详情、附件下载、SAFE 浮动入口 | 微信授权、访客画像 | H5 已明确公开访问 |
| 政策内容 | 查询、新增、编辑、查看、删除、富文本、附件 | 草稿、审核、下架 | 当前要求保存后立即展示 |
| 金融产品 | 查询、新增、编辑、查看、删除、Excel 导入校验、首批预置 112 条 | H5 搜索和高级筛选 | 首版可通过栏目直接浏览 |
| 后台账号 | 登录、权限控制、人员管理、重置密码、本人修改密码 | 第三方登录 | 账号密码登录满足当前场景 |
| 操作日志 | 记录关键新增、修改、删除、导入和账号管理动作 | 复杂审计分析和报表 | 首版只需要操作追溯 |
| 固定配置 | 17 项银行机构、14 个发布办公室、2 类产品、4 个县域服务队、SAFE 地址和 Logo | 银行机构、服务队后台维护 | 当前固定写入即可 |
| 附件存储 | 服务器本地文件目录、配置化路径、备份 | 对象存储 | 首版部署更简单 |
