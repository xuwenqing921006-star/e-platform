# 央行 E 平台业务 PRD 对齐稿

## 目标
- 建设面向微信公众号入口的公开 H5 页面，用于展示后台发布的政策内容与银行金融产品。
- 建设配套 WEB 管理后台，让不同单位账号按权限维护内容，并支持管理员统一管理。

## 范围
- 包含：微信公众号 H5、WEB 管理后台、账号权限、政策内容管理、附件下载、金融产品管理、Excel 批量导入、人员管理、操作日志。
- 不包含：微信授权登录、访客信息采集、草稿或下架状态、H5 搜索筛选、银行名单后台维护、县域金融业务服务队后台维护。

## 业务场景
### 场景 1：公众浏览金融服务
- 用户从微信公众号进入 H5，无需登录。
- 用户可切换查看服务指引、政策宣传和助企通道。
- 服务指引和政策宣传按发布时间倒序展示，进入详情后可阅读正文并下载附件。
- 助企通道混合展示涉农产品和小微产品，列表与详情展示产品所属分类和银行机构。

### 场景 2：公众浏览乡村振兴内容
- 用户切换到乡村振兴后，可选择肇州县、肇源县、林甸县或杜蒙县。
- 每个县展示本县服务指引和金融业务服务队信息。
- 服务指引由对应县域账号发布；服务队介绍和成员信息固定写入 H5。

### 场景 3：后台发布政策内容
- 管理员可管理所有单位发布的内容。
- 普通账号固定绑定一个单位，仅能管理本单位发布的内容。
- 四个县域账号只能发布服务指引。
- 内容新增或编辑保存后立即在 H5 展示；删除后立即移除。
- 每篇内容最多上传 3 个附件，单个附件不超过 20 MB，支持 PDF、Word 和 Excel 文件。

### 场景 4：后台管理金融产品
- 仅管理员和货币信贷政策管理科账号可以管理金融产品。
- 金融产品包含产品名称、所属银行、分类、准入条件、产品简介、业务经办人和联系电话。
- 产品分类固定为涉农产品和小微产品。
- 所属银行从固定下拉列表选择，不提供后台维护入口。
- 支持按模板批量导入，导入前校验错误行。

### 场景 5：后台管理人员和查看日志
- 仅管理员可以新增、查看、修改、删除账号以及重置密码。
- 普通账号只能修改自己的密码。
- 仅管理员可以查看简洁操作日志，日志记录操作人、操作时间、操作类型和操作对象。

## 验收口径
- H5 可公开访问，栏目切换与详情展示符合已确认结构。
- 市级和县域服务指引能根据发布办公室自动进入对应栏目。
- 后台权限隔离正确，普通账号不能跨单位管理内容。
- 首次上线预置参考 Excel 中的 112 条金融产品，其中涉农产品 52 条、小微产品 60 条。
- 产品批量导入按工作表自动标记分类，且能提示错误行。
- 右下角数字外管平台浮动按钮可展开、关闭和跳转，详情页不展示。

## 已确认的特殊规则
- Excel 中 `阳光惠农贷` 特殊行的所属银行和产品名称均使用 `阳光惠农贷`，其余内容按原表顺延整理。
- 银行固定下拉项严格按参考 Excel 的“银行机构”列去重，不自行修正名称。
