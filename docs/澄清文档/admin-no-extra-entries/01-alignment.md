# 后台原型外入口清理业务 PRD 对齐稿

## 目标
- 后台继续使用若依管理端样式，但可见入口、字段和模块必须收敛到央行 E 平台已确认业务范围。

## 范围
- 包含：H5 列表卡片边界、H5/后台已导入中文数据乱码、后台顶部工具入口、默认若依系统菜单和个人中心多余字段。
- 不包含：后台像素级还原 Pencil 原型；不替换若依管理端整体视觉框架。

## 业务场景
场景1：H5 用户浏览服务指引和产品列表
- 用户在乡村振兴或金融服务栏目查看卡片列表。
- 卡片内容必须在移动端容器内正常换行，不应横向撑破页面。
- 标题、机构、产品名称等中文内容必须正常显示。

场景2：后台用户进入管理端
- 用户登录后台后仍看到若依风格的管理端外壳。
- 顶部不展示全局搜索、全屏、布局大小、消息通知、锁屏、个人中心、布局设置等原型外入口。
- 用户下拉菜单只保留修改密码和退出登录。

场景3：后台业务菜单
- 侧边栏只展示已确认的业务入口：内容管理、金融产品、账号管理、操作日志。
- Excel 导入作为产品管理的隐藏路由保留，不作为独立侧边栏入口。
- 若依默认的系统管理、系统监控、系统工具、通知公告、代码生成等入口不在业务后台可见。

## 验收口径
- H5 卡片不横向溢出，中文列表内容正常显示。
- 后台菜单和顶部区域不出现上述原型外入口。
- 后台可见业务页面仍可访问并使用若依/Element UI 风格。

## 需要对齐的问题
- 无。用户已确认“就用若依的样式”，同时本次明确要求删除原型外无用信息。
