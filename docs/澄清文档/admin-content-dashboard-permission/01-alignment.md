# 后台内容发布与首页快捷入口权限口径

## 背景

用户验收后台时发现两个权限相关体验问题：

1. 非管理员进入新增内容页面时，仍可选择发布办公室。
2. 工作概览的快捷操作在用户没有对应功能权限时仍可点击，点击后进入无权限路由并显示 404。

## 已确认口径

- 后台新增或编辑内容时，如果当前登录用户不是管理员，发布办公室默认选中该用户绑定的办公室。
- 非管理员的发布办公室不可修改。
- 管理员仍可选择任意发布办公室。
- 工作概览的快捷操作必须按当前用户按钮权限判断可用性。
- 用户没有对应功能权限时，快捷操作直接不可点击，不允许点击后再进入 404。

## 验收标准

- 普通办公室账号点击“新增内容”后，办公室字段自动显示本人办公室，并处于禁用状态。
- 管理员点击“新增内容”后，办公室字段可正常选择。
- 没有 `centralbank:product:add` 或 `centralbank:product:import` 的用户，在首页对应快捷入口为不可点击状态。
- 有权限的快捷入口仍可正常跳转。
