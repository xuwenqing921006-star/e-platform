## 5. 发现的问题

### P0-1 H5 附件下载链接线上会失败

现象：

- 后端返回的附件地址是 `/api/public/attachments/{id}/download`。
- 线上访问 `https://xczx.hljfinance.com/api/public/attachments/{id}/download` 返回 404。
- 同一个附件改为 `https://xczx.hljfinance.com/h5/api/public/attachments/{id}/download` 可正常下载。
- 后台内容详情弹窗中的“下载”链接同样使用 `/api/public/attachments/{id}/download`，UI 点击链路复测也返回 404。

影响：

- H5 文章详情中的附件链接如果直接使用后端 `download_url`，用户点击会下载失败。
- 后台发布后查看详情时，管理员点击附件“下载”也会失败。

建议：

- 优先修 Nginx：放通 `/api/public/attachments/*` 到后端 `/api/public/attachments/*`。
- 或前端/H5 对附件 `download_url` 做前缀转换，把 `/api/` 转成 `/h5/api/`。
- 更长期建议：后端附件公开 URL 支持按部署前缀配置，不硬编码根 `/api`。

### P1-1 `/h5` 不带斜杠返回 404

现象：

- `https://xczx.hljfinance.com/h5/` 正常。
- `https://xczx.hljfinance.com/h5` 返回 404。

影响：

- 用户手动输入或外部链接少写末尾 `/` 时打不开。

建议：

- Nginx 增加精确重定向：`/h5 -> /h5/`。

### P1-2 根路径 `/` 重定向不合理

现象：

- `https://xczx.hljfinance.com/` 返回 302。
- `Location` 指向 `http://xczx.hljfinance.com:82/admin/`。

影响：

- 从 HTTPS 根域跳到 HTTP 和 82 端口，不符合正式部署入口习惯。
- 可能造成移动端、微信内置浏览器或政企网络访问异常。

建议：

- 根路径统一重定向到正式入口，例如 `/h5/` 或 `/admin/`。
- 不要在公网入口暴露 `:82` 端口跳转。
- 保持 HTTPS。

### P1-3 H5 图片放大层关闭按钮可见但点击被原图拦截

现象：

- 临时文章插入富文本图片后，H5 详情页点击图片可以打开原图查看器。
- `.article-image-viewer-close` 关闭按钮存在且可见。
- Playwright 点击关闭按钮时，事件被 `<img alt="正文图片原图">` 拦截，关闭失败。

影响：

- 手机用户点击关闭按钮可能不稳定，尤其是大图覆盖按钮区域时。

建议：

- 提高关闭按钮层级，确保按钮在图片之上。
- 关闭按钮使用固定定位和明确 `z-index`。
- 检查图片查看器内部图片是否覆盖了关闭按钮的点击区域。
- 补一条 E2E 测试：点击富文本图片 -> 原图层打开 -> 点击关闭 -> 原图层消失。

### P1-4 账号姓名仍不是期望值“总管理员”

- 用户之前期望改为“总管理员”。

影响：

- 后台右上角昵称和账号管理列表仍与预期不一致。

建议：

- 核对生产库 `admin` 账号姓名字段。
- 如果以后允许管理员在账号管理里改自己姓名，需要确认 UI、`getInfo`、账号列表三处数据来源一致。

### P1-5 退出登录后跳转到 404 页面

现象：

- 后台右上角点击“退出登录”并确认后，浏览器跳转到 `https://xczx.hljfinance.com/index`。
- `/index` 线上返回 404。
- 再访问 `https://xczx.hljfinance.com/admin/index` 时，路由守卫会跳回 `/admin/login?redirect=%2Findex`，说明 token 已清理，但退出落地页错误。

影响：

- 用户退出后看到 404，会误以为系统异常。

建议：

- 调整前端退出登录跳转路径到 `/admin/login`。
- 如果保留 RuoYi 默认 `/index`，需要在 Nginx 或前端路由中确保 `/admin/index` 和 `/index` 的 base path 一致；当前部署在 `/admin/` 子路径下，更建议统一跳 `/admin/login`。

### P1-6 SAFE 外链在移动端可能显示“数字外管平台打开失败”

现象：

- H5 首页点击右下角 SAFE 浮动入口，展开后点击“国家外汇管理局数字外管平台”。
- 部分手机浏览器或微信内置浏览器中会显示“数字外管平台打开失败，请稍后重试。”。
- 该提示来自前端组件 `H5SafeEntry.vue`，不是数字外管平台外站直接返回的错误。

定位：

- 当前组件点击外链时执行 `event.preventDefault()`，阻止了 `<a>` 标签原生跳转。
- 随后通过 `window.open(SAFE_PORTAL_URL, '_blank')` 打开新窗口。
- 如果移动端、微信内置浏览器或浏览器安全策略拦截新窗口，`window.open` 会返回空值，前端就显示失败提示。
- 当前 SAFE 地址为 `http://zwfw.safe.gov.cn/asone/`，正式 H5 是 HTTPS；HTTPS 页面打开 HTTP 外链且外链还会 302 到政务登录 SSO，更容易触发移动端限制。

影响：

- 用户会误以为数字外管平台不可用。
- 实际可能只是新窗口打开被拦截，或 HTTP 外链/SSO 跳转被当前浏览器环境限制。

建议：

- 移动端 H5 优先使用原生 `<a href target="_blank" rel="noopener noreferrer">` 跳转，不要手动 `preventDefault` + `window.open`。
- 或在 `window.open` 失败时兜底到 `window.location.href = SAFE_PORTAL_URL`，使用当前页跳转。
- 建议优先改为当前页跳转，兼容微信内置浏览器和普通手机浏览器。
- 修改后补充 E2E：展开 SAFE 入口 -> 点击外链 -> 不显示失败提示，并能触发跳转行为。

### P2-1 内容附件数量超过 3 个时返回 500

现象：

- 提交 4 个 `attachment_ids`，返回 HTTP 200 + `code=500`，消息是“每篇内容最多上传 3 个附件”。

影响：

- 前端虽然能看到错误消息，但接口语义不规范；自动化、网关、监控会把它当成功 HTTP 请求或服务端异常。

建议：

- 后端把该类业务校验统一返回 HTTP 400 + `code=400`。
- 增加控制器测试覆盖“附件超过 3 个”的 HTTP 状态码。

### P2-2 账号姓名超长返回数据库截断异常

现象：

- 编辑账号时提交超长 `display_name`，返回 HTTP 200 + `code=500`，底层为 MySQL `DataTruncation`。

影响：

- 普通用户看到服务端异常，不知道应如何修正。
- 存在数据库字段长度与前后端校验不一致的问题。

建议：

- 前端账号姓名输入框增加最大长度限制。
- 后端 DTO/Service 增加长度校验，返回 HTTP 400。
- 视业务需要扩容数据库字段。

### P2-3 账号用户名超长返回数据库截断异常

现象：

- 创建账号时使用超过数据库 `sys_user.user_name` 实际长度的用户名，返回 HTTP 200 + `code=500`。
- 底层异常为 MySQL `DataTruncation: Data too long for column 'user_name'`。
- 前端账号输入框当前 `maxlength=40`，但数据库字段实际更短。

影响：

- 账号新增会出现服务端异常，用户无法得到明确“用户名过长”的业务提示。

建议：

- 前端登录账号最大长度与数据库字段统一。
- 后端创建账号前增加用户名长度校验，返回 HTTP 400 + 明确提示。
- 同步检查 `.env`、建表 SQL、测试数据和账号导入/创建文档中的用户名长度约束。

- `https://xczx.hljfinance.com/prod-api/login` 返回 404。
- 当前线上实际可用前缀是 `/h5/api` 和 `/admin/prod-api`。
- 2026-07-04 补测“后台-金融产品”的模板下载、Excel 导入、新增产品时再次确认：
  - `POST https://xczx.hljfinance.com/prod-api/login` 返回 404。
  - `POST https://xczx.hljfinance.com/admin/prod-api/login` 返回 200，并可继续完成产品模板下载、新增产品、Excel 校验导入、提交导入、导入后查询与测试数据清理。

影响：

- 如果文档、旧前端、附件 URL 或外部系统使用根 `/api`、`/prod-api`，会失败。
- 如果部署版后台前端仍以 `/prod-api` 作为 `VUE_APP_BASE_API`，后台登录、产品模板下载、Excel 导入、新增产品等接口都会在正式环境失败；只有统一走 `/admin/prod-api` 或 Nginx 同时代理根 `/prod-api` 才能稳定。

建议：

- 明确正式部署只支持 `/h5/api`、`/admin/prod-api`，并同步文档。
- 如果后端返回的公开 URL 仍是 `/api`，必须补根 `/api` 代理。
- 确认正式构建产物中的后台 API base 是否为 `/admin/prod-api`；如果仍为 `/prod-api`，需要重新构建后台或在 Nginx 增加 `/prod-api` 到后端的代理规则。
